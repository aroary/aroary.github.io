About

This tool is for sending single messages to a webhook. It supports sending embeds and customizing your username and profile picture. It also supports sending a custom JSON request to your webhook. Messages are sent from your computer, not a server that is serving this website.

Creating a webhook

In your discord server, open a channel settings and open the integrations tab (Channel > Edit > Integrations) or open the integrations tab for the server (Server Settings > integrations). Select View Webhooks then New Webhook. Now you can set the default name and profile picture for your webhook and change the channel. The default name and profile will be overridden if you specify a Name and Profile URL here.

Sending a message

To send a message, paste your webhook url into the Webhook URL input. Then, optionally add a webhook username in Name and a webhook profile picture in Profile url. Now you need to add content to your data to send. In the content input, add a message up to 2000 characters long. To send your message to your webhook, Press the send button. Note that when you send a message your input value will be cleared.

Adding an embed

To add an embed, click the embed button to open the embed inputs. Fill in the data in the appropriate locations. Inputs the specify URL must be http or https protocols. Note that if you are sending an embed you do not need to include any message content in the Content input.

Using JSON

To add custom JSON, press the JSON button to enable JSON format. A JSON format input must include at least either a content parameter or an embeds parameter. Example minimum JSON: {content:"DATA"}.

URL Parameters

To set the parameters automatically when you open the tab, you can use url parameters. To set a specific parameter, at the end of the url add ?name=value with the name the parameters name and value the parameters value you want to set it to be.

Parameter names:
  • Webhook URL: webhook
  • Profile URL: avatar
  • Username: name
  • Content: content
  • Author: author
  • Author Icon URL: authorIcon
  • Title: title
  • Description: description
  • Color: color
  • URL: url
  • Image URL: image
  • Thumbnail URL: thumbnail
  • Footer Value: footer
  • Footer Icon URL: footerIcon

Errors

If the request status is 204 then your request succeded. If your request status is 400 then your inputs are invalid and need to be corrected. Inputs are not reset if the status is an error.
